threadql

v0.1.0

Multi-tenant natural language to SQL via Slack. ThreadQL uses LLMs to convert user questions into SQL queries, execute them against configured datasources, and return formatted results through Slack.

Installation

# Add the chart repository
helm repo add emtay https://emtay-com.github.io/helm-charts
helm repo update
# Install with default values
helm install threadql emtay/threadql
# Install with custom values
helm install threadql emtay/threadql -f my-values.yaml

Included Components

⚙ App (PHP-FPM + Nginx)
Laravel application with PHP 8.4-FPM and Nginx reverse proxy in a single pod
⚡ Worker
Supervisord-managed queue workers for background job processing
🔒 SSH Tunnel
Secure tunnel manager for remote database connectivity
🗃 Redis
In-cluster Redis for queues, cache, and sessions
Optional — can use external Redis
🗂 MySQL 8.0
In-cluster MySQL for application database with persistent storage
Optional — can use external database
🌐 Ingress
Configurable ingress with HAProxy or Traefik support
🔒 cert-manager
Optional Let's Encrypt TLS via ClusterIssuer
Requires cert-manager pre-installed
🕐 CronJobs
Scheduled schema scans and maintenance tasks
🚀 Migration Job
Helm hook runs database migrations before install/upgrade
